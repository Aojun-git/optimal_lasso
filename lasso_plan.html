最优化课程展示 第6/7/8章综合 Python手写迭代

LASSO 回归问题的多种优化算法对比：从非光滑建模到复合优化求解

推荐题目：“LASSO 问题的多算法实现与收敛行为对比”

核心目标：用同一个 LASSO 模型串联第 6 章梯度/次梯度思想、第 7 章增广拉格朗日/ADMM 思想、第 8 章近端梯度、FISTA、坐标下降等复合优化算法；用手写循环实现算法，并用图表展示收敛速度、稀疏恢复效果、参数敏感性和实际预测表现。

1. 选题定位与展示主线

一句话主线：LASSO 是“光滑二次损失 + 非光滑 L1 正则”的典型复合优化问题。它既能体现建模意义，也能自然比较不同最优化算法的思想差异：直接次梯度、近端梯度、Nesterov 加速、坐标下降、ADMM。
\[ \min_x F(x)=\frac{1}{2}\|Ax-b\|_2^2+\mu\|x\|_1 \]
课程章节对应 在展示中的作用 建议讲解深度
第6章：无约束优化算法
梯度下降、次梯度思想		 作为基线：说明 L1 在 0 点不可微，直接用次梯度法可以做，但收敛慢、震荡明显。 中等：重点讲“为什么朴素方法不够好”。
第7章：约束优化算法
增广拉格朗日思想		 通过变量分裂 \(x=z\) 得到 ADMM，展示“把难问题拆成两个容易子问题”的思路。 中等：不做复杂证明，强调原始残差/对偶残差。
第8章：复合优化算法
ISTA/FISTA、坐标下降、ADMM		 展示主体：LASSO 最自然属于复合优化，近端算子软阈值是核心。 重点：公式、伪代码、收敛曲线和实验对比都围绕这一章展开。
推荐展示结论：FISTA 通常在目标函数下降上快于 ISTA；坐标下降在中小规模、特征数不太大时实现简单且效果稳定；ADMM 在变量分裂视角下解释性强，适合展示约束优化思想；次梯度法最容易实现，但作为 LASSO 主求解器通常不是最优选择。

2. 问题建模：为什么 LASSO 值得讲

2.1 从最小二乘到稀疏回归

普通最小二乘只追求拟合误差小：

\[ \min_x \frac{1}{2}\|Ax-b\|_2^2 \]

当特征很多、样本较少、或者希望模型具有可解释性时，我们更希望解向量 \(x\) 中只有少数非零元素。LASSO 加入 L1 正则：

\[ \min_x \frac{1}{2}\|Ax-b\|_2^2+\mu\|x\|_1 \]

其中 \(A\in \mathbb{R}^{m\times n}\) 是样本-特征矩阵，\(b\in \mathbb{R}^{m}\) 是观测值，\(x\in \mathbb{R}^n\) 是待估计参数，\(\mu>0\) 控制稀疏程度。

2.2 需要回答的三个问题

建模问题
L1 为什么能带来稀疏性？\(\mu\) 越大，解为什么越稀疏？
算法问题
L1 不可微时，为什么不能直接照搬普通梯度下降？
实验问题
不同算法在目标函数、恢复误差、运行时间、稀疏度上各有什么差异？

2.3 展示中建议保留的直观图

图片含义制作方式
L1/L2 几何示意图 L1 球有尖角，更容易与约束线在坐标轴附近相交，因此更容易产生稀疏解。 自己用 matplotlib 画二维菱形、圆和直线。
真实稀疏 \(x^\star\) vs 恢复 \(\hat{x}\) 展示 LASSO 对稀疏信号恢复的效果。 stem plot 或 bar plot。
\(\mu\) 路径图 展示不同正则化强度下，非零系数数量和预测误差如何变化。 x 轴为 \(\mu\)，y 轴为非零个数/测试 MSE。

3. 算法模块设计

建议实现 5 个算法，其中 4 个作为正式对比，1 个作为“为什么需要更好算法”的基线。所有算法都用 Python + NumPy 手写 for/while 循环，不调用 sklearn 的 Lasso、scipy.optimize.minimize、cvxpy 等求解器。

3.1 公共工具函数

函数作用备注
objective(A,b,x,mu) 计算 \(F(x)=0.5\|Ax-b\|^2+\mu\|x\|_1\) 所有算法共用，便于画收敛曲线。
grad_smooth(A,b,x) 计算光滑项梯度 \(A^T(Ax-b)\) ISTA/FISTA/次梯度法使用。
soft_threshold(v,lam) 软阈值算子 \(S_\lambda(v)=\mathrm{sign}(v)\max(|v|-\lambda,0)\) LASSO 的核心近端算子。
metrics(...) 计算目标值、恢复误差、非零个数、测试误差、运行时间 统一生成表格。

3.2 算法一：次梯度法 Subgradient Descent（基线）

把 L1 项看成不可微项，用次梯度近似处理：

\[ g_k=A^T(Ax_k-b)+\mu\cdot s_k,\quad s_{k,i}\in \partial |x_{k,i}| \]
初始化 x = 0
for k in range(max_iter):
    grad = A.T @ (A @ x - b)
    subgrad_l1 = sign(x)              # x_i=0 时可取 0，作为简单实现
    step = alpha0 / sqrt(k + 1)       # 衰减步长更稳
    x = x - step * (grad + mu * subgrad_l1)
    记录 objective、误差、稀疏度

展示意义：它最贴近第 6 章次梯度思想，代码非常简单，但通常目标下降慢、稀疏性不够干净，可以作为反衬。

3.3 算法二：ISTA / 近似点梯度法

将 LASSO 写为 \(f(x)+g(x)\)，其中 \(f(x)=0.5\|Ax-b\|^2\) 可微，\(g(x)=\mu\|x\|_1\) 不可微但有简单近端算子。

\[ x_{k+1}=S_{\mu/L}\left(x_k-\frac{1}{L}A^T(Ax_k-b)\right) \]
估计 L = 最大特征值(A.T @ A) 或使用 power iteration
初始化 x = 0
for k in range(max_iter):
    y = x - (1/L) * A.T @ (A @ x - b)
    x = soft_threshold(y, mu/L)
    记录 objective、误差、稀疏度

展示意义：这是 LASSO 的主算法之一，体现“梯度步 + 近端修正”的复合优化思想。

3.4 算法三：FISTA / Nesterov 加速近端梯度

FISTA 在 ISTA 基础上加入动量点 \(y_k\)，通常能显著加快目标函数下降。

\[ x_{k+1}=S_{\mu/L}\left(y_k-\frac{1}{L}A^T(Ay_k-b)\right),\quad t_{k+1}=\frac{1+\sqrt{1+4t_k^2}}{2} \]
\[ y_{k+1}=x_{k+1}+\frac{t_k-1}{t_{k+1}}(x_{k+1}-x_k) \]
初始化 x = 0, y = x, t = 1
for k in range(max_iter):
    x_old = x.copy()
    x = soft_threshold(y - (1/L) * A.T @ (A @ y - b), mu/L)
    t_new = (1 + sqrt(1 + 4*t*t)) / 2
    y = x + ((t - 1) / t_new) * (x - x_old)
    t = t_new
    记录 objective、误差、稀疏度

展示意义：对应第 8 章 Nesterov 加速/FISTA，可通过半对数收敛曲线直观看出“加速”的作用。

3.5 算法四：坐标下降法 Coordinate Descent

逐个更新 \(x_j\)，固定其他坐标。令残差 \(r=b-Ax\)，第 \(j\) 个坐标更新可写为：

\[ \rho_j=a_j^T(r+a_jx_j),\quad x_j \leftarrow \frac{S_{\mu}(\rho_j)}{\|a_j\|_2^2} \]
初始化 x = 0, r = b - A @ x
for epoch in range(max_epoch):
    for j in range(n):
        r = r + A[:, j] * x[j]              # 加回旧贡献
        rho = A[:, j].T @ r
        x_new_j = soft_threshold(rho, mu) / (norm(A[:, j])**2)
        r = r - A[:, j] * x_new_j          # 扣除新贡献
        x[j] = x_new_j
    记录 objective、误差、稀疏度

展示意义：对应第 8 章分块坐标下降法。它不需要全局矩阵求逆，也非常适合展示“逐坐标解析更新”。

3.6 算法五：ADMM / 交替方向乘子法

做变量分裂：令 \(x=z\)，把二次损失和 L1 正则拆开：

\[ \min_{x,z}\frac{1}{2}\|Ax-b\|_2^2+\mu\|z\|_1,\quad s.t.\ x-z=0 \]

ADMM 的 scaled form 更新为：

\[ x^{k+1}=(A^TA+\rho I)^{-1}(A^Tb+\rho(z^k-u^k)) \]
\[ z^{k+1}=S_{\mu/\rho}(x^{k+1}+u^k),\quad u^{k+1}=u^k+x^{k+1}-z^{k+1} \]
预计算 M = A.T @ A + rho * I
初始化 x,z,u = 0
for k in range(max_iter):
    # 不直接求 inverse；可用 np.linalg.solve 解线性方程
    x = solve(M, A.T @ b + rho * (z - u))
    z_old = z.copy()
    z = soft_threshold(x + u, mu/rho)
    u = u + x - z
    primal_res = norm(x - z)
    dual_res = rho * norm(z - z_old)
    记录 objective、残差、误差、稀疏度

展示意义：对应第 7/8 章的增广拉格朗日与交替方向乘子法，适合展示“约束化/分裂化”的思想。

实现注意：ADMM 中可以使用 np.linalg.solve 解线性方程，但不要用 np.linalg.inv 显式求逆；这不算依赖求解包，而是基础线性代数操作。若想更“手写”，可额外实现一个简单 Cholesky 或共轭梯度版本作为附录，不建议作为主任务，工作量会明显增加。

4.3 数据生成与加载伪代码

合成稀疏数据

def make_synthetic_lasso(m=128, n=256, k=25, noise=0.01, corr=0.0, seed=0):
    rng = np.random.default_rng(seed)

    # 1. 生成设计矩阵 A
    A = rng.normal(size=(m, n)) / np.sqrt(m)

    # 可选：构造相关特征，模拟病态问题
    if corr > 0:
        for j in range(1, n):
            A[:, j] = corr * A[:, j-1] + np.sqrt(1 - corr**2) * A[:, j]

    # 2. 生成 k-稀疏真实向量 x_star
    x_star = np.zeros(n)
    support = rng.choice(n, size=k, replace=False)
    x_star[support] = rng.normal(size=k)

    # 3. 生成观测 b
    b = A @ x_star + noise * rng.normal(size=m)

    # 4. 标准化，避免不同特征尺度影响算法比较
    A = standardize_columns(A)
    b = b - b.mean()
    return A, b, x_star, support

8. 最终希望得到的结论框架

可以预期的结论

  • 相比直接次梯度法，近端方法更适合 LASSO 这种“光滑 + 非光滑”的复合结构。
  • FISTA 在目标函数下降速度上通常优于 ISTA，但可能存在轻微震荡。
  • 坐标下降法实现直观，在特征规模中等时表现稳定。
  • ADMM 的变量分裂解释性强，可以通过原始残差和对偶残差观察约束满足过程。
  • \(\mu\) 越大，解越稀疏；但过大可能导致欠拟合，测试误差上升。

需要避免的风险

  • 不要只画迭代次数曲线，还要画 CPU 时间曲线，否则不同算法单步成本不可比。
  • 不要只做真实数据，因为没有真实 \(x^\star\)，无法展示“恢复误差”。
  • 不要把代码写成直接调用现成 LASSO 求解器，否则不符合课程展示要求。
  • 不要过度讲复杂证明，展示重点应放在“算法思想 + 手写实现 + 实验现象”。

10. 可能被问到的问题与回答口径

问题建议回答
为什么 L1 能产生稀疏性，而 L2 不行？ L1 球有尖角，优化解更容易落在坐标轴附近；从算法上看，软阈值会把小系数直接压到 0，而 L2 通常只是连续缩小系数。
为什么不用普通梯度下降？ L1 在 0 点不可微，普通梯度不适用。可以用次梯度，但效率较低；近端梯度能利用 L1 的特殊结构，更新后直接得到稀疏解。
FISTA 为什么比 ISTA 快？ FISTA 在近端梯度基础上加入 Nesterov 动量，利用历史方向构造外推点，因此目标下降通常更快。
ADMM 和 ISTA 的本质区别是什么？ ISTA 直接在原变量上做“梯度步 + 近端步”；ADMM 先引入 \(x=z\)，把二次项和 L1 项拆开，交替优化两个子问题，并用乘子更新约束一致性。
如何选择 \(\mu\)？ 从建模角度，\(\mu\) 控制拟合误差与稀疏性之间的权衡；实验中可以画 \(\mu\) path，用验证集误差和非零个数共同选择。
